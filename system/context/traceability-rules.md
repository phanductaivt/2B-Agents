# Traceability Rules

## Core Link Chain
- Requirement Input
- BRD
- FRS
- User Story
- Acceptance Criteria
- Feature List
- Wireframe
- UI
- Review Notes
- Test Cases

## Required Traceability Files
- Requirement-level matrix
- Requirement-level Mermaid flow
- Project-level summary

## Rules
- Always include requirement ID and artifact IDs.
- Use file existence checks for coverage status.
- Mark status as `Complete`, `Partial`, or `Missing`.
- Keep mappings explicit (not checkmarks only).
