# Output Standards

## Core Requirement Outputs
- `clarification.md`: facts, assumptions, questions, actions.
- `process-bpmn.md`: Mermaid process flow.
- `user-story.md`: story and dependencies.
- `acceptance-criteria.md`: testable criteria.
- `brd.md`: business goals, scope, rules, benefits.
- `frs.md`: functional requirements and flows.
- `feature-list.md`: hierarchical feature breakdown.
- `wireframe.md`: UX structure and screen notes.
- `ui.html`: FE demo page.
- `review-notes.md`: validator findings and status.
- `test-cases.md`: linked test case matrix.
- `artifact-status.md`: artifact-by-artifact status, gate, and approval.
- `artifact-checklist.md`: practical checklist per artifact.
- `gate-report.md`: current gate state and blocked artifacts.
- `artifact-reviews/*.md`: artifact-specific review file with checklist result, issues, and next action.

## Flow/Traceability Outputs
- `requirement-flow.md`
- `requirement-traceability-matrix.md`
- `requirement-traceability-flow.md`
- project-level `project-flow.md`
- project-level `requirement-traceability-summary.md`

## Formatting Rules
- Markdown-first for all documents except `ui.html`.
- Keep sections short and business-readable.
- Include linked IDs in relevant files.
- Approval states must use one of: Draft, In Review, Approved, Rework Needed, Blocked.
