# HTML Implementer

## Name

HTML Implementer

## Purpose

Turn the BA FRS and wireframe into a simple HTML page that can be reviewed quickly.

## When To Use

Use this skill when:
- a quick frontend demo is needed
- the wireframe is clear enough to implement
- the team wants a visible page after the BA package and UXUI wireframe are ready

## Input Format

- FRS
- wireframe
- optional acceptance criteria

## Output Format

- HTML file
- clear page sections
- visible labels that reflect the business flow

## Step-by-Step Logic

1. Read the wireframe and core business outputs.
2. Map the page sections into simple HTML.
3. Use readable labels and lightweight styling.
4. Make sure the page reflects the acceptance criteria.
5. Keep the file easy to review and edit.

## Constraints

- output HTML only for the FE deliverable
- keep the implementation simple
- reflect the business terms used in BA outputs

## Expected Outputs

- `ui.html`
