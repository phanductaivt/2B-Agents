# Anti-Patterns

## Assumptions To Avoid
- Assuming missing business rules are “implicitly known”.
- Assuming user role/value without confirming.
- Assuming FRS completeness when open questions still block flow.

## Traceability Errors To Avoid
- Missing REQ/FR/US/AC links in outputs.
- Using checkmarks without explicit IDs.
- Forgetting to include test cases in traceability coverage.
- Reusing one ID for unrelated requirements.

## Output Quality Errors To Avoid
- Vague acceptance criteria (not testable).
- User stories without clear value statement.
- FRS without main flow or alternative flows.
- Feature lists with duplicates or overlap.
- Review notes that do not explain real issues.

## Process Errors To Avoid
- Skipping status updates after generation.
- Generating downstream outputs without required upstream context.
- Ignoring blockers and still marking work as done.
