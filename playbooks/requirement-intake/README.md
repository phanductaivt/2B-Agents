# Requirement Intake Playbook

This playbook helps the team move from one project requirement input file to a richer BA package.

Use it when:
- a stakeholder sends rough requirement notes
- a BA needs a clear first draft
- a product stakeholder wants a small package for review before backlog refinement

Main output files:
- `clarification.md`
- `process-bpmn.md`
- `user-story.md`
- `acceptance-criteria.md`
- `brd.md`
- `frs.md`
- `feature-list.md`
- `review-notes.md`

The selected project controls:
- where the input file lives
- which project config is used
- which project knowledge is loaded first
- where the final output is written
